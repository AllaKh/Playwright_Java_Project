Booking fails with client‑side exception when dates are unavailable

Summary:
Attempting to access /reservation/<roomId> with a date range that is unavailable causes the entire application to throw a client‑side exception and render an error banner instead of a graceful “Room unavailable” message.

Description:
Instead of returning a user‑friendly notice, the SPA crashes and shows:
“Application error: a client-side exception has occurred while loading automationintesting.online (see the browser console for more information).”
All page functionality is lost until the user manually navigates away.

Steps to Reproduce:

  1. Open
    https://automationintesting.online/reservation/2?checkin=2020-01-01&checkout=2020-01-02 (any obviously unavailable range will work).
  2. Observe the screen after the SPA finishes loading.

Actual Result:
A full‑screen error banner appears; DevTools console shows an uncaught JavaScript exception. The booking UI does not load.

Expected Result:
User should receive a clear “No availability for selected dates” message and be able to choose different dates without the application crashing.

Severity: Critical
The bug blocks the primary booking flow and crashes the entire front‑end.

Issue 1: Phone field accepts letters instead of digits only

Summary:
Phone input field allows letters and non-numeric characters, but should restrict input to digits only.

Description:
Currently, the phone number input field in the contact form accepts alphabetic characters and other non-numeric symbols. This leads to invalid phone numbers being submitted and may cause issues during data processing or backend validation. The field should strictly enforce numeric input only (digits 0-9).

Steps to Reproduce:

  1. Navigate to the contact form page.
  2. Enter letters (e.g., abcde) or mixed characters (e.g., 123abc) in the phone number field.
  3. Submit the form.

Actual Result:
The form accepts phone numbers containing letters and does not display any validation error.

Expected Result:
The phone field should accept only digits. Attempting to enter letters or special characters should trigger a validation error message such as "Phone must contain digits only" and prevent form submission.

Severity: Medium

Cannot update or delete an existing booking – client‑side crash

Summary:
When a user attempts either to update or delete a booking, the application throws a client‑side exception and displays the generic error banner instead of completing the requested action.

Description:
Both operations that should modify an existing reservation lead to a complete front‑end failure. The page shows the message:
“Application error: a client-side exception has occurred while loading automationintesting.online (see the browser console for more information).”
The user cannot proceed further; a refresh only reloads the same error.

Steps to Reproduce (Update flow):

  1. Create a new booking (the flow works and returns a confirmation).
  2. Click the pencil/edit icon (or “Update” button) next to the booking.
  3. Try to change any field and press Reserve Now.

Steps to Reproduce (Delete flow):

  1. Create a new booking.
  2. Press the red Delete (trash) button.

Actual Result (both flows):
The SPA crashes and renders the client‑side exception banner; DevTools console shows an uncaught JavaScript error.

Expected Result:

  • Update: Booking details should be saved and a success banner shown.
  • Delete: Booking should be removed and the empty booking form should re‑appear without errors.

Severity: Critical
The defect breaks essential CRUD functionality—users cannot edit or cancel their reservations, potentially causing over‑booking and support escalations.
